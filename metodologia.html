Metodología

Pycods utiliza una investigación de tipo aplicada y descriptiva, con un enfoque mixto (cualitativo y cuantitativo), desarrollada bajo la metodología ágil Scrum. Esto permitió organizar el trabajo en fases con entregas progresivas, identificar errores fácilmente y agregar funcionalidades de forma ordenada.

Fases de Desarrollo

Fase 1

Planificación

Se definió la arquitectura modular de Pycods, organizando el código en carpetas como Core, Models y Widgets. Se analizaron los requerimientos funcionales y no funcionales del sistema, identificando las necesidades de accesibilidad, gamificación e interacción para niños con TEA.

Fase 2

Desarrollo de Contenido

Se desarrollaron los módulos de aprendizaje de Python con temas básicos: variables, print(), condicionales if, bucles for y listas. El contenido es visual e interactivo, adaptado para los niños con TEA. Se integraron ejercicios prácticos con retroalimentación automática para facilitar la comprensión y motivación.

Fase 3

Gamificación e Interfaz

Se implementó el sistema de gamificación con monedas, logros y minijuegos educativos. La interfaz web fue diseñada con HTML, CSS y Bootstrap 5, usando JavaScript para animaciones e interacciones, con PHP como backend. Se utilizaron widgets reutilizables para optimizar la experiencia del usuario con adaptación visual.

Fase 4

Integración de Servicios

Se integró PHP con MySQLi para la gestión segura de usuarios y autenticación mediante sesiones PHP. Se utilizó MySQL como base de datos relacional para almacenar el progreso de cada estudiante, monedas y avances dentro de la plataforma. La sincronización en tiempo real facilita la persistencia de datos y el seguimiento del proceso de aprendizaje.
